## NK long-read downstream workflow

This folder contains a simple downstream workflow for the PacBio Kinnex NK data.
The scripts are numbered and meant to be run in order.

### Main idea

The point of the long-read data is not to repeat short-read clustering.
The useful part is transcript structure:

- differential isoform usage within genes
- recurrent novel isoforms across donors
- alternative TSS usage
- alternative polyA site usage
- exploratory population-level RNA variant calling

### Order

1. `01_split_mapped_bams_by_population.sh`
2. `02_build_shared_isoform_catalog.R`
3. `03_make_population_pseudobulks.R`
4. `04_test_isoform_usage.R`
5. `05_rank_novel_isoforms.R`
6. `06_test_tss_polya_usage.R`
7. `07_call_population_variants.sh`
8. `08_plot_longread_results.R`

### Notes

- Step 2 is important. The donor-level PacBio transcript IDs are not directly comparable across donors.
- The scripts therefore build a shared catalog from transcript structure before any cross-donor testing.
- The statistical scripts use donor-matched comparisons, which is the right level of replication here.
- RNA variant calls from step 7 should be treated as supportive and inspected manually.
